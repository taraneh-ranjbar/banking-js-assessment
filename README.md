# 💳 Banking JavaScript Assessment

A practical JavaScript project focused on core concepts used in real-world applications such as data transformation, functional programming, and object composition.

---

## Overview

This project simulates a simple banking system where account data is processed, validated, and extended using modern JavaScript patterns.

The goal is not just to solve problems, but to write **clean, predictable, and maintainable code**.

---

## Concepts Covered

* Pure Functions (immutability, predictable outputs)
* Functional Programming (`map`, `filter`, `reduce`)
* Context Handling (`this`, `bind`)
* Higher-Order Functions
* Currying & Partial Application
* Object Composition (role-based capabilities)
* Prototype vs Instance methods

No dependencies required.

```bash
node banking_Sara.js
```

## Key Highlights

* Avoids mutation of original data
* Uses modern ES6+ features
* Focuses on readability over clever tricks
* Demonstrates real-world problem solving (not just theory)

---

## Example Features

* Format and sanitize account data safely
* Process accounts using reusable functions
* Handle asynchronous context issues (`this` problem)
* Build flexible employee roles using composition
* Create reusable fee calculators using currying

## Why This Project Matters

This project reflects real-world scenarios where:

* Data integrity is critical (e.g. banking systems)
* Code must be predictable and testable
* Flexibility is preferred over rigid class hierarchies
